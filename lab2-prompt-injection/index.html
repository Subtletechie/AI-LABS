OBJECTIVE: Execute 3 distinct tokenization attacks
Select Attack Scenario
⚡ Unicode Homoglyph

Character Substitution

Replace ASCII chars with visually identical Unicode. Bypasses keyword filters that operate pre-tokenization.

⚡ Zero-Width Injection

Invisible Character Splitting

Insert zero-width spaces to split tokens the filter expects as whole units. Filter sees "admin" as two tokens.

⚡ Token Boundary Exploit

Instruction Fragmentation

Split malicious instructions across token boundaries. Each fragment is benign; the assembled instruction is not.

⚡ Context Window Attack

Token Budget Exhaustion

Craft inputs that expand disproportionately during tokenization, consuming context window or triggering truncation.

Level 4 · Defense Build
Build and validate token-aware defenses against the Level 3 attack techniques. Each defense layer targets a specific attack class — stack them to build depth.
OBJECTIVE: Apply all 4 defense layers
Target Input (from attacker)
Defense Stack — Apply Layers
Unicode Normalization (NFKC)

Normalize all Unicode to NFKC form before tokenization. Collapses homoglyphs and fullwidth chars to ASCII equivalents. Kills homoglyph substitution attacks.

Zero-Width Character Stripping

Strip U+200B, U+200C, U+200D, U+FEFF, and related invisible characters. Prevents zero-width injection from splitting expected tokens.

Token Budget Enforcement

Pre-tokenize input and reject if token count exceeds threshold relative to character count. Detects expansion attacks and repetition-based budget exhaustion.

Post-Normalization Semantic Scan

After normalization, run pattern matching on the cleaned string. Catches fragmented instructions that reconstruct after defense layers 1–3 are applied.
