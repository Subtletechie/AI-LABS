Stage 1 Data Ingestion

Correct! Row 7 has a feature_score of 0.02 — far below any other row — yet its label is 1. This inconsistency is a classic data poisoning signature. An attacker who controls the data source can insert rows like this to influence how the model learns.

Stage 4 Model Artifacts

A model file is just bytes. Without a cryptographic hash, you have no way to know if those bytes were modified after training — by an attacker, a storage error, or a supply-chain substitution.
model_v1.pkl — ORIGINAL
📦
model_v1.pkl — COPY
📦
⚠ One byte changed
What does this tell you?
Not quite. Different training would produce a different model file from the start. Here, we're comparing the same file before and after storage — the hash mismatch happened after training, which means the change occurred to the artifact itself, not during training.
Exactly. Even a single changed byte produces a completely different SHA-256 hash. This is what makes cryptographic hashing so powerful for integrity verification. The original and tampered files are distinguishable instantly.
The hash function works correctly — that's exactly why the mismatch was detected. SHA-256 reliably produces different outputs for different inputs. The issue is with the file, not the function.
Hashes don't change on their own. A SHA-256 hash is deterministic: the same bytes always produce the same hash. If the hash changed, the underlying bytes changed. That change demands investigation.

Stage 5 Evaluation & Approval

A model that scores 91% on standard tests can still be dangerous. Adversarial test sets probe for hidden weaknesses — biases, backdoors, edge cases. Both gates must pass before a model is safe to deploy.
Standard Test Accuracy 0%
PASS ✓
Adversarial Test Accuracy 0%
FAIL ✗ (threshold: 70%)
Should this model be approved for deployment?
Not safe. The adversarial accuracy of 68% is below the required 70% threshold. A model that fails adversarial testing may behave unpredictably on edge cases, adversarial inputs, or inputs that differ from the training distribution. Both gates must pass.
Correct. An approval gate requires ALL checks to pass. 91% standard accuracy looks good — but the 68% adversarial score reveals a weakness that could be exploited. The model needs more work before deployment.
