Stage 4 Model Artifacts

A model file is just bytes. Without a cryptographic hash, you have no way to know if those bytes were modified after training — by an attacker, a storage error, or a supply-chain substitution.
model_v1.pkl — ORIGINAL
📦
model_v1.pkl — COPY
📦
⚠ One byte changed
What does this tell you?
Different training would produce a different model from the start. The hash mismatch happened after training — the file itself changed.
Exactly. Even a single changed byte produces a completely different SHA-256 hash. The original and tampered files are distinguishable instantly.
The hash function works correctly — that's why the mismatch was detected. The issue is with the file, not the function.
Hashes don't change on their own. SHA-256 is deterministic: the same bytes always produce the same hash. A changed hash means changed bytes.
